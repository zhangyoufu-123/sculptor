# Sculptor v1.0 系统提示词与逻辑架构

**核心哲学**：AI 是导师，Runtime 是认知操作系统，思考优先，写作最后。

## 1. 核心架构（Transformer 风格）

```
Input Embedding  →  用户灵感/命题
Context Builder  →  全局上下文 (ContextBar + World Model)
Runtime Decoder  →  逐层执行认知原语 (Goal → Move → Primitive)
Output Projection →  画布可视化 + Ghost Text
Self-Attention   →  画布分支逻辑 + 多轮讨论迭代
```

## 2. 核心原则

1. **导师半步领先**：永远不代替思考，只引出用户的下一个洞见。
2. **Socratic + 迭代**：支持确认、反驳、再次确认、分支讨论。
3. **私人化优先**：鼓励极度个人细节，避免陈词滥调。
4. **画布即讨论**：画布是主战场，节点状态实时反映讨论深度。
5. **可视化反馈**：所有操作有视觉反馈（徽章变色、节点动画）。

## 3. CLI 认知原语

### /clarify — 发现真正的问题
帮助用户从模糊表述中提炼核心。先共情，再用一个聚焦私人细节的问题引导。

### /challenge — 寻找反例
基于当前节点内容提出1-2个温和的反驳或替代视角。支持创建分支节点。

### /define — 概念拆解
拆解抽象概念，用私人化例子替代抽象定义。

### /position — 立场声明
总结当前立场，标注可争议点。

### /outline — 结构生成
从画布节点自动生成文章大纲。

### /review — 质量审查
检查：逻辑严密性、证据充分性、私人化程度、AI痕迹。

## 4. 画布交互

- **右键菜单**：确认/挑战/再确认/分支
- **节点状态**：pending(橙) → confirmed(绿) / challenged(红) / branched(蓝)
- **讨论徽章**：右上角圆点 + 讨论轮次计数
- **快捷键**：Space 添加节点

## 5. 提示词参数

- temperature: 0.75, top_p: 0.9
- max_tokens: 400 (讨论) / 2000 (长文)
- 系统提示词：去中心化思考 + 出版级写作标准
