# Sculptor EchoWall v6.0 — 写作伴随系统设计

> 基于用户11轮深度访谈 + Jenni AI 竞品分析
> 日期: 2026-07-09

---

## 一、核心理念

回声壁不是"信息面板"，而是**安静读稿的编辑 + 随时待命的助手 + 偶尔抬头的灯塔**。

- 用户写作时，回声壁始终在"读"——不打扰，但一直在
- 用户停顿时，回声壁递上灵感——不强制，随手可用
- 用户迷失时，回声壁用犀利问题拉回来

## 二、三层数据引擎（技术壁垒）

按优先级排列：

### 第1层：历史风格库（首选）

```
数据来源：
  ├── 用户已写文章 → 段落切片 → 向量化存储
  ├── 用户 AI 对话历史 → 风格偏好 → 关键词提取
  ├── 用户采纳/拒绝记录 → 风格指纹更新
  └── 用户本地文档 → 全文索引

检索方式：
  ├── 语义相似度匹配（当前段落 vs 历史段落）
  ├── 风格指纹匹配（语气/节奏/用词偏好）
  └── 主题关键词匹配

用途：
  ├── 检索相似写作场景下用户用过的比喻/金句
  ├── 匹配用户偏好的论证模式（三段论/先破后立/层层递进）
  └── 提供"你上次写类似内容时用了这个句式"
```

### 第2层：AI 实时创作（次选）

```
触发条件：
  ├── 停笔 3-5 秒 → 环境通道建议
  ├── 双击文段/框选 → 意图通道分析
  └── 段落完成 → 诊断卡刷新

输出类型：
  ├── 比喻句/诗句/例子（知识注入）
  ├── 段落续写（风格匹配的 Ghost Text 增强版）
  ├── 逻辑质问（犀利提问打破死循环）
  └── 段落回放 + 读者视角提示（整段理解）
```

### 第3层：通用知识库（补强）

```
数据来源：
  ├── 中国古诗词库（匹配主题的诗句/典故）
  ├── 名人名言库（中英文，按主题索引）
  ├── 修辞手法模板库（比喻/排比/反问/设问）
  └── 论证模式库（Toulmin/Classic/Rogerian 框架）

用途：
  ├── 当历史库无匹配时回退
  ├── AI 创作时作为 prompt 增强上下文
  └── 深度搜索（网络检索实时补充）
```

## 四、两个交互通道

### 通道一：环境通道（始终在线）

```
触发：写作过程中持续运行
节奏：
  ├── 段落完成后 → 诊断卡全量刷新（平滑过渡，不跳不闪）
  ├── 停笔 3-5 秒 → 灵感流追加新条目
  └── 实时微提醒 → 静默更新（仅图标闪烁，不弹跳）

内容来源顺序：
  1. 历史风格库检索 → 有匹配则显示
  2. 无匹配 → AI 实时生成
  3. AI 无置信 → 通用知识库回退

用户行为：
  ├── 不采纳 → 继续写，建议自然滚出视野
  ├── 采纳 → 点击复制/插入
  └── 反馈 → 👍/👎 影响后续推荐权重
```

### 通道二：意图通道（用户主动触发）

```
触发方式：
  ├── 双击文段 → 分析该段逻辑结构
  └── 框选文字 → 针对选中内容生成建议

触发后行为：
  ├── 诊断卡切换为"深度分析模式"
  │   ├── 逻辑结构可视化（前提→论据→结论）
  │   ├── 薄弱点标注
  │   └── 改进建议（3条以内）
  └── 灵感流切换为"针对性推荐"
      ├── 相似场景的历史文本
      ├── AI 生成的改写版本
      └── 可替换的词汇/句式
```

## 五、节奏控制引擎

```
段落检测：
  └── 检测两个连续换行 → 触发段落分析
      ├── 提交当前段落文本给 AI
      ├── 同时检索历史风格库
      └── 1-2 秒后刷新诊断卡

停顿检测：
  └── 最后输入后 3 秒无操作
      ├── 触发灵感流更新
      └── 光标处显示微弱"..."提示

微提醒：
  └── 实时规则检测（不依赖 AI）
      ├── 重复词检测（同一个词出现超过阈值）
      ├── 段落长度异常（过长/过短）
      ├── 逻辑连接词缺失
      └── 直接在诊断卡静默更新

平滑过渡：
  └── CSS transition 0.4s ease
  ├── 旧内容 opacity → 0
  ├── 新内容 opacity → 1
  └── 无跳动、无闪烁
```

## 七、技术决策

1. **历史风格库**：使用 `transformers.js` 本地向量化（不需要服务器），存储在 IndexedDB，用户隐私零泄露
2. **AI 调用**：复用现有 DeepSeek 客户端，新增 `POST /api/echo/analyze` 和 `POST /api/echo/inspire` 两个端点
3. **通用知识库**：纯本地 JSON 索引 + 网络搜索回退，不依赖外部 API
4. **段落检测**：ProseMirror plugin，监听 `doc` 变化，检测连续 `\n\n`
5. **状态管理**：新增 `useEchoWall` hook，独立于现有 Zustand store
6. **不在编辑器内直接显示建议**（与 Ghost Text 区分）：回声壁的所有内容都在侧边栏，用户选择采纳后才进入编辑器
